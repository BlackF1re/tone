# OpenWRT Tone Generator

Утилита для воспроизведения звуков (тонов) на роутерах с OpenWRT через встроенный баззер.

## Описание

Программа `tone` позволяет генерировать звуки на роутере путём управления GPIO пином, подключённым к баззеру (пьезоэлектрическому звукоизлучателю). Программа запускается с параметрами частоты и длительности.

### Основные характеристики

- Управление баззером через sysfs интерфейс GPIO
- Real-Time приоритет (SCHED_FIFO) для точного управления частотой
- Активное ожидание (busy wait) для высокоточного контроля временных интервалов
- Минимальные зависимости (только стандартная библиотека C)

## Использование

```bash
./tone <frequency> <duration_ms>
```

**Параметры:**
- `<frequency>` — частота звука в Герцах (Hz). Значение ≤ 0 означает молчание (пауза)
- `<duration_ms>` — длительность в миллисекундах

**Примеры:**
```bash
./tone 1000 500      # Звук 1 кГц на 500 мс
./tone 440 1000      # Звук ля (A4) на 1 секунду
./tone 0 2000        # Пауза на 2 секунды
```

## Поиск пути к баззеру в системе

Путь к баззеру зависит от модели роутера и типа GPIO контроллера. В данном коде используется путь:
```
/sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0/gpio/buzzer/value
```

### Процесс поиска правильного пути

#### 1. Найти все GPIO контроллеры

На роутере выполните:
```bash
ls -la /sys/class/gpio/
```

Или поищите по всему /sys:
```bash
find /sys -name "gpiochip*" -type d
```

Результат может выглядеть так:
```
/sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0
```

#### 2. Найти GPIO пины

Посмотрите, какие GPIO пины доступны:
```bash
ls /sys/class/gpio/
# или
ls /sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0/gpio/
```

#### 3. Определить пин баззера

Проверьте документацию вашего роутера или ищите пины с названием `buzzer`:
```bash
find /sys -name "*buzzer*"
```

Или ищите по содержимому директорий GPIO:
```bash
ls /sys/devices/platform/*/gpio/*/
```

#### 4. Найти интерфейс для управления

Когда найдёте пин баззера, путь для управления будет:
```
/sys/devices/platform/.../gpio/buzzer/value
```

Проверьте, что файл доступен для записи:
```bash
cat /sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0/gpio/buzzer/value
```

Если вы видите 0 или 1 — это правильный путь. Если файл не существует, ищите другой путь по инструкции выше.

#### 5. Альтернативный поиск через /proc и /sys

Проверьте содержимое `/proc/device-tree/` (если доступно):
```bash
find /proc/device-tree -name "*buzzer*"
```

Или посмотрите на pinctrl/GPIO привязку в дереве устройств:
```bash
cat /proc/device-tree/gpio-buzzer/gpios
```

## Сборка на роутере

### Способ 1: Прямая сборка на роутере (если установлены tools)

1. **Передайте исходник на роутер:**
   ```bash
   scp tone.c root@192.168.1.1:/tmp/
   ```

2. **Подключитесь к роутеру по SSH:**
   ```bash
   ssh root@192.168.1.1
   ```

3. **Установите gcc (если не установлен):**
   ```bash
   opkg update
   opkg install gcc libc-dev
   ```

4. **Скомпилируйте:**
   ```bash
   cd /tmp
   gcc -O2 -o tone tone.c -lrt
   ```

   Флаги компиляции:
   - `-O2` — оптимизация для скорости
   - `-lrt` — линковка с библиотекой real-time (для `clock_gettime`)

5. **Проверьте сборку:**
   ```bash
   ./tone 1000 500
   ```

### Способ 2: Кросс-компиляция (рекомендуется)

Если на роутере нет инструментов разработки, используйте OpenWRT SDK для кросс-компиляции:

1. **Скачайте OpenWRT SDK** для вашего роутера с [официального сайта](https://openwrt.org/downloads)

2. **Распакуйте SDK:**
   ```bash
   tar xzf OpenWrt-SDK-*.tar.xz
   cd OpenWrt-SDK-*
   ```

3. **Скопируйте tone.c в SDK:**
   ```bash
   cp tone.c ./
   ```

4. **Скомпилируйте с инструментами SDK:**
   ```bash
   ./staging_dir/toolchain-*/bin/*-gcc -O2 -o tone tone.c -lrt
   ```

5. **Передайте бинарник на роутер:**
   ```bash
   scp tone root@192.168.1.1:/usr/bin/
   chmod +x /usr/bin/tone  # на роутере
   ```

### Способ 3: Через Makefile

Создайте Makefile для удобства:

```makefile
CC ?= gcc
CFLAGS ?= -O2 -Wall
LDFLAGS = -lrt

all: tone

tone: tone.c
	$(CC) $(CFLAGS) -o tone tone.c $(LDFLAGS)

clean:
	rm -f tone

install: tone
	cp tone /usr/bin/tone
	chmod +x /usr/bin/tone

.PHONY: all clean install
```

Затем:
```bash
make
make install
```

## Требования

- **ОС:** OpenWRT / Linux с поддержкой sysfs GPIO
- **Библиотеки:** libc, librt (стандартные)
- **Разрешения:** root доступ для изменения приоритета (sched_setscheduler)

## Примечания

- **Активное ожидание (busy wait):** Программа использует активное ожидание вместо sleep для максимальной точности частоты. Это может потребить значительные ресурсы процессора.
- **Real-Time приоритет:** Использование SCHED_FIFO требует root доступа.
- **Путь к баззеру:** Путь, указанный в коде, специфичен для определённых моделей роутеров. Для других моделей может потребоваться поиск корректного пути.
