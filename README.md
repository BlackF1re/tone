# OpenWRT Tone Generator

Утилита для воспроизведения звуков (тонов) на роутерах с OpenWRT через встроенный баззер.

## Описание

Программа `tone` позволяет генерировать звуки на роутере путём управления GPIO пином, подключённым к баззеру (пьезоэлектрическому звукоизлучателю). Программа запускается с параметрами частоты и длительности.

### Основные характеристики

- Управление баззером через sysfs интерфейс GPIO
- Real-Time приоритет (SCHED_FIFO) для точного управления частотой
- Активное ожидание (busy wait) для высокоточного контроля временных интервалов
- Минимальные зависимости (только стандартная библиотека C)

## Использование

```bash
./tone <frequency_hz> <duration_ms> [options]
```

**Аргументы:**
- `<frequency_hz>` — частота звука в Герцах (Hz). Диапазон: 1-20000 Hz, значение ≤ 0 означает молчание (пауза)
- `<duration_ms>` — длительность в миллисекундах (максимум 300000 = 5 минут)

**Опции:**
- `-d, --device PATH` — путь к GPIO файлу баззера (если не указан, автоматически определяется)
- `-p, --pwm VOLUME` — PWM контроль громкости (0-100%, по умолчанию 100%)
- `-c, --chirp END_HZ` — линейный свип частоты от START к END Hz
- `-v, --verbose` — включить подробный вывод
- `-vv` — очень подробный вывод со статистикой (jitter, ошибки)
- `-h, --help` — показать справку

**Примеры:**
```bash
# Базовый звук
./tone 1000 500                  # Звук 1 кГц на 500 мс
./tone 440 1000                  # Звук ля (A4) на 1 секунду
./tone 0 2000                    # Пауза на 2 секунды

# PWM контроль громкости (требует настройки оборудования)
./tone 2500 300 -p 50            # 50% мощности
./tone 2500 300 -p 25            # 25% мощности (тихо)

# Chirp (частотная модуляция)
./tone 1000 500 -c 2000          # Свип от 1 кГц до 2 кГц за 500 мс
./tone 500 1000 -c 5000          # Сирена: от 500 Гц до 5 кГц

# Подробный вывод и диагностика
./tone 2000 500 -v               # Verbose mode
./tone 2000 500 -vv              # С полной статистикой (jitter, overhead)
./tone 2000 500 -vv -d /sys/class/gpio/gpio10/value
```

**Автоматическое определение баззера:**

Программа автоматически ищет баззер в стандартных местах:
- `/sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0/gpio/buzzer/value`
- `/sys/devices/platform/10000000.palmbus/10000600.gpio/gpiochip0/gpio/buzzer/value`
- `/sys/class/gpio/buzzer/value`
- `/sys/class/gpio/gpio0/value` ... `/sys/class/gpio/gpio4/value`

Если автоопределение не сработало, вы можете явно указать путь:
```bash
./tone 2000 500 -d /sys/class/gpio/gpio12/value
```

Или через переменную окружения:
```bash
export BUZZER_GPIO="/sys/class/gpio/gpio10/value"
./tone 1000 500
```

**Сборка с нестандартным путём:**

Если вам известен точный путь на вашем роутере, вы можете собрать программу с жёстко задананным путём (эффективнее для производства):

```bash
make clean
make CFLAGS="-O2 -Wall -Wextra -std=c99 -DBUZZER_GPIO_PATH=\"/sys/class/gpio/gpio10/value\""
# или
gcc -O2 -DBUZZER_GPIO_PATH=\"/path/to/buzzer\" -c tone.c
gcc -o tone tone.o
```

При сборке с определённым путём программа не будет искать баззер, а сразу его использовать.

## Поиск пути к баззеру в системе

Путь к баззеру зависит от модели роутера и типа GPIO контроллера. В данном коде используется путь:
```
/sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0/gpio/buzzer/value
```

### Процесс поиска правильного пути

#### 1. Найти все GPIO контроллеры

На роутере выполните:
```bash
ls -la /sys/class/gpio/
```

Или поищите по всему /sys:
```bash
find /sys -name "gpiochip*" -type d
```

Результат может выглядеть так:
```
/sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0
```

#### 2. Найти GPIO пины

Посмотрите, какие GPIO пины доступны:
```bash
ls /sys/class/gpio/
# или
ls /sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0/gpio/
```

#### 3. Определить пин баззера

Проверьте документацию вашего роутера или ищите пины с названием `buzzer`:
```bash
find /sys -name "*buzzer*"
```

Или ищите по содержимому директорий GPIO:
```bash
ls /sys/devices/platform/*/gpio/*/
```

#### 4. Найти интерфейс для управления

Когда найдёте пин баззера, путь для управления будет:
```
/sys/devices/platform/.../gpio/buzzer/value
```

Проверьте, что файл доступен для записи:
```bash
cat /sys/devices/platform/1e000000.palmbus/1e000600.gpio/gpiochip0/gpio/buzzer/value
```

Если вы видите 0 или 1 — это правильный путь. Если файл не существует, ищите другой путь по инструкции выше.

#### 5. Альтернативный поиск через /proc и /sys

Проверьте содержимое `/proc/device-tree/` (если доступно):
```bash
find /proc/device-tree -name "*buzzer*"
```

Или посмотрите на pinctrl/GPIO привязку в дереве устройств:
```bash
cat /proc/device-tree/gpio-buzzer/gpios
```

## Сборка на роутере

### Способ 1: Прямая сборка на роутере (если установлены tools)

1. **Передайте исходник на роутер:**
   ```bash
   scp tone.c root@192.168.1.1:/tmp/
   ```

2. **Подключитесь к роутеру по SSH:**
   ```bash
   ssh root@192.168.1.1
   ```

3. **Установите gcc (если не установлен):**
   ```bash
   opkg update
   opkg install gcc libc-dev
   ```

4. **Скомпилируйте:**
   ```bash
   cd /tmp
   gcc -O2 -o tone tone.c -lrt
   ```

   Флаги компиляции:
   - `-O2` — оптимизация для скорости
   - `-lrt` — линковка с библиотекой real-time (для `clock_gettime`)

5. **Проверьте сборку:**
   ```bash
   ./tone 1000 500
   ```

### Способ 2: Кросс-компиляция (рекомендуется)

Если на роутере нет инструментов разработки, используйте OpenWRT SDK для кросс-компиляции:

1. **Скачайте OpenWRT SDK** для вашего роутера с [официального сайта](https://openwrt.org/downloads)

2. **Распакуйте SDK:**
   ```bash
   tar xzf OpenWrt-SDK-*.tar.xz
   cd OpenWrt-SDK-*
   ```

3. **Скопируйте tone.c в SDK:**
   ```bash
   cp tone.c ./
   ```

4. **Скомпилируйте с инструментами SDK:**
   ```bash
   ./staging_dir/toolchain-*/bin/*-gcc -O2 -o tone tone.c -lrt
   ```

5. **Передайте бинарник на роутер:**
   ```bash
   scp tone root@192.168.1.1:/usr/bin/
   chmod +x /usr/bin/tone  # на роутере
   ```

### Способ 3: Через Makefile

Используйте Makefile для удобной сборки и установки:

```bash
make                # Собрать бинарник
make install        # Установить в /usr/bin (требует root)
make clean          # Удалить объектные файлы
make help           # Показать справку
```

**Для кросс-компиляции:**
```bash
make CC=arm-openwrt-linux-gcc    # Используйте правильный компилятор для вашего роутера
```

**Установка в нестандартное место:**
```bash
make install INSTALL_DIR=/tmp    # Установить в /tmp вместо /usr/bin
```

## Требования

- **ОС:** OpenWRT / Linux с поддержкой sysfs GPIO
- **Библиотеки:** libc, librt (стандартные)
- **Разрешения:** root доступ для:
  - Доступа к GPIO (для записи в `/sys/devices/...`)
  - Установки Real-Time приоритета (SCHED_FIFO)

### 🔍 Автоопределение пути

```bash

```
**Как это работает:** вместо полной амплитуды используется модуляция ширины импульса (PWM) на несущей частоте 10 кГц. Это уменьшает среднюю мощность, но сохраняет тоновый сигнал.

### 🔄 Chirp (частотная модуляция)
```bash
./tone 1000 500 -c 2000   # Линейный свип от 1 кГц до 2 кГц
./tone 500 1000 -c 5000   # Сирена-подобный эффект
```
Полезно для создания звуковых сигналов, диагностических звуков, звуков навигации.

### 📊 Расширенная статистика (-vv)
```
--- Statistics ---
Total toggles: 45000
Total time: 500.123 ms
Min period: 499 ns
Max period: 501 ns
Jitter: 2 ns
```
Помогает диагностировать проблемы с точностью и стабильностью.

### 🛡️ Сохранение состояния GPIO
- Программа **сохраняет исходное состояние GPIO** перед началом
- **Восстанавливает его после завершения**
- Гарантирует, что не нарушится работа других систем, использующих этот GPIO

### ✅ Проверка доступности
- **Проверка без тестовых писков** через `access(path, W_OK)`
- **Громкие ошибки** вместо молчания если баззер недоступен
- Правильные сообщения об ошибках в stderr

### ⏱️ Таймауты и защита
- Максимальная длительность: **5 минут** (300000 мс)
- Предотвращает зависание и перегрузку системы
- Контроль частоты в диапазоне 1-20000 Hz

## ⚡ Быстрые примеры (одна команда)

Самые интересные звуки, которые можно воспроизвести прямо в командной строке:

### Базовые сигналы
```bash
# Классический звуковой сигнал (школьный звонок)
tone 1000 300

# Поднимающийся звук (ракета или волшебство)
tone 1000 0 -c 4000

# Опускающийся звук (грустный результат)
tone 4000 0 -c 1000

# Двойной сигнал (две ноты)
tone 1500 150 && tone 2000 150

# Низкий гудок (сигнал готовности)
tone 200 500

# Высокий писк (восклицание)
tone 5000 100

# Микс громкости (громко + тихо + тишина)
tone 2000 100 -p 100 && tone 2000 100 -p 30 && tone 0 100

# Тревога полусекундной длительности
tone 3000 500 -p 80

# Проверка звука (4 разных частоты)
for f in 1000 2000 3000 4000; do tone $f 100 -p 80; done

# Весёлый "ping" (как в компьютерных играх)
tone 3000 50 -p 100 && sleep 0.1 && tone 3000 50 -p 100
```

### Мелодии и музыкальные звуки
```bash
# Мелодия "Марш" (военный марш)
tone 330 150 -p 100 && tone 330 150 && tone 330 150 && tone 264 200

# Звук успеха (3 подряд идущих ноты вверх)
tone 400 100 && tone 500 100 && tone 600 100

# Звук неудачи (опускающиеся ноты)
tone 600 100 && tone 500 100 && tone 400 100

# Первые ноты гимна
tone 440 200 && tone 494 200 && tone 523 200

# Колокольный звон (высокие частоты)
tone 8000 200 && tone 7000 150 && tone 6000 150

# Музыкальная лестница вверх (8 нот)
for f in 262 294 330 349 392 440 494 523; do tone $f 100; done

# Звук "да-да-да-дааа" (от Битлов)
tone 400 100 && tone 400 100 && tone 400 100 && tone 200 300
```

### Специальные эффекты
```bash
# Фейерверк (быстрые звуки вверх и вниз)
for i in 1 2 3; do tone 1000 0 -c 5000; tone 5000 0 -c 1000; done

# Взрыв (низкая частота, нарастающий объём)
tone 100 300 -p 20 && tone 100 200 -p 50 && tone 100 100 -p 100

# Волна или сирена (медленный chirp)
tone 1000 1000 -c 5000

# UFO приближение (волны нарастающие)
tone 2000 100 && tone 2500 100 && tone 3000 100 && tone 3500 100

# UFO отдаляется (волны нисходящие)
tone 3500 100 && tone 3000 100 && tone 2500 100 && tone 2000 100

# Звук телепортации (быстрый взлёт и падение)
tone 1000 0 -c 8000 && tone 8000 0 -c 1000

# Лазер (быстрый звук вверх)
tone 1000 200 -c 5000

# Выстрел (быстрое нарастание и спад)
tone 500 50 && tone 0 30
```

### Звуки оборудования и устройств
```bash
# Звук загрузки (3 ступеньки)
tone 800 100 && tone 1200 100 && tone 1600 100

# Звук выключения (опускающийся звук)
tone 2000 300 -c 200

# Звук включения (поднимающийся звук)
tone 200 300 -c 2000

# Печать/сканирование (ритмичные звуки)
for i in 1 2 3 4; do tone 1500 50; sleep 0.1; done

# Кодовый замок открыт (успешный ввод)
tone 1200 100 && tone 1500 100 && tone 1800 100

# Кодовый замок ошибка (резкий низкий звук)
tone 300 200
```

### Ритмические паттерны
```bash
# Сердцебиение (два быстрых звука + пауза)
for i in 1 2 3 4; do tone 1000 50 && tone 1200 50; sleep 0.3; done

# Маячный ритм (один длинный + два коротких)
tone 2000 300 && tone 3000 100 && tone 3000 100

# Тик-так часов
tone 1000 100 && sleep 0.2 && tone 1200 100

# Morse код (точка-точка-тире): SOS
tone 1000 100 && sleep 0.1 && tone 1000 100 && sleep 0.1 && tone 1000 100 && sleep 0.3 && tone 1000 300

# Звон будильника (повторяющиеся пары)
for i in 1 2 3 4 5; do tone 2000 50 && tone 0 20 && tone 2000 50 && sleep 0.1; done
```

## 📝 Примеры скриптов для роутера

### 1. Простой звуковой сигнал

```bash
#!/bin/sh
# alarm.sh - Простое будильник

echo "Alarm!"
tone 2000 100
tone 0 50
tone 2000 100
```

### 2. Полиция-сирена

```bash
#!/bin/sh
# police_siren.sh - Классическая сирена

for i in 1 2 3 4 5; do
    tone 2000 100 -p 100   # Высокий звук на полную мощность
    tone 0 30              # Короткая пауза
    tone 1000 100 -p 100   # Низкий звук
    tone 0 30              # Пауза
done
```

### 3. Звуковой сигнал с понижением (тревога)

```bash
#!/bin/sh
# alert.sh

tone 3000 200 -c 1000  # Chirp вниз от 3кГц до 1кГц
sleep 1
tone 2500 200 -c 1000  # Повтор
```

### 4. Громкость регулируется от переменной

```bash
#!/bin/sh
# volume_control.sh

VOLUME=${1:-100}  # Громкость из параметра, по умолчанию 100%

if [ "$VOLUME" -lt 0 ] || [ "$VOLUME" -gt 100 ]; then
    echo "Volume must be 0-100"
    exit 1
fi

echo "Playing at ${VOLUME}% volume..."
tone 2500 500 -p "$VOLUME"
```

### 5. Мелодия через последовательные частоты

```bash
#!/bin/sh
# melody.sh - Простая мелодия

# Ноты: C=262, D=294, E=330, F=349, G=392, A=440, B=494

play_note() {
    freq=$1
    duration=$2
    tone "$freq" "$duration"
}

# Мелодия "Twinkle Twinkle Little Star"
play_note 262 300  # C
play_note 262 300  # C
play_note 392 300  # G
play_note 392 300  # G
play_note 440 300  # A
play_note 440 300  # A
play_note 392 600  # G

```


## ⚠️ Важные замечания

1. **Баззеры разные** — одна программа может звучать по-разному на разных роутерах. Экспериментируйте!

2. **PWM требует поддержки** — на некоторых роутерах PWM может не работать или работать неправильно. Тестируйте с `-vv`!

3. **Real-time приоритет** — попытаемся установить, но это может не работать. Программа продолжит работу с обычным приоритетом.

4. **Питание** — мощные звуки (частые переключения) потребляют больше энергии. Используйте PWM для снижения нагрузки.

5. **Безопасность** — не используйте слишком часто и слишком громко, это может повредить оборудование!
